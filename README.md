Лабораторная работа: Задача "Читатель-Писатель" (kthread vs pthread)
Цель — реализовать классическую задачу синхронизации "Читатели-Писатели" в пространстве ядра Linux (с помощью kthread) и в пространстве пользователя (с помощью pthread).
Структура проекта

HW_07_kthread/
├── Makefile      # Скрипт для сборки модуля и программы
├── rw_kernel.c   # Исходный код модуля ядра
├── app_us.c      # Исходный код программы в userspace
└── README.md     # (Этот файл)

1. Сборка проекта

Перейдите в директорию проекта и выполните команду для сборки всего проекта:

make all

Эта команда скомпилирует:

    Модуль ядра (rw_kernel.ko).
    Программу в userspace (app_us).

2. Запуск модуля ядра (Kernel Space)

    Загрузите модуль в ядро:
    bash
    Копировать

    sudo make load

    Вы увидите в системном логе (dmesg) сообщения от потоков читателей и писателей.
    Дайте модулю поработать некоторое время (например, 15 секунд):
    bash
    Копировать

    sleep 15

    Выгрузите модуль:
    bash
    Копировать

    sudo make unload

    В логе (dmesg) появится финальное сообщение с итоговым значением счетчика.

3. Запуск программы в userspace (User Space)

Просто выполните скомпилированный бинарный файл:
bash
Копировать

./app_us

Программа будет работать, пока счетчик не достигнет предельного значения. Вы увидите вывод от потоков в терминале.

Цели сборки (Makefile)

В проекте реализованы все необходимые цели make для удобства работы:

    make или make all: Собрать модуль ядра и userspace-программу.
    make load: Загрузить модуль ядра (insmod rw_kernel.ko).
    make unload: Выгрузить модуль ядра (rmmod rw_kernel).
    make app_us: Собрать только userspace-программу.
    make clean: Удалить все скомпилированные файлы (.ko, .o, исполняемые файлы).
    make format: Отформатировать исходный код (требуется установленный clang-format).
    make check: Вывести список всех доступных целей сборки для проверки.
